This repository contains the **Infrastructure as Code (IaC)** and **GitOps Manifests** for the Personality Simulation Engine project. It manages cloud resources (DigitalOcean) and the Kubernetes Cluster configuration.

## 🏗 Repository Structure

-   **[`terraform/`](./terraform)**: Terraform configuration files for:
    -   DigitalOcean Kubernetes (DOKS)
    -   Managed PostgreSQL Database
    -   Private Container Registry
    -   Kubernetes Secrets and Helm (ArgoCD, NGINX Ingress)
-   **[`gitops-manifests/`](./gitops-manifests)**: The Kubernetes YAML manifests that define:
    -   The Backend-API Deployment and Service.
    -   The Frontend-WEB Service and Ingress rules.
    -   ArgoCD applications for GitOps-style continuous delivery.

## 🚀 Deployment Guide

### 1. Terraform Setup
1.  Enter the directory: `cd terraform`
2.  Create `terraform.tfvars`:
    ```hcl
    do_token = "your_digitalocean_token_here"
    ```
3.  Deploy Infrastructure:
    ```bash
    terraform init
    terraform plan
    terraform apply
    ```

### 2. Connect to Kubernetes
Once Terraform completes, retrieve your credentials:
```bash
doctl kubernetes cluster kubeconfig save main-cluster
```

### 3. GitOps Flow (ArgoCD)
ArgoCD is installed automatically by the `argocd.tf` file. 

#### 🔐 Accessing ArgoCD
1.  **Retrieve Admin Password**:
    ```powershell
    # In PowerShell (Windows)
    kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | % { [System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String($_)) }
    ```
2.  **Expose the Dashboard**:
    ```bash
    kubectl port-forward svc/argocd-server -n argocd 8080:443
    ```
3.  **Login**: Visit [https://localhost:8080](https://localhost:8080) and use username `admin` with the password retrieved above.

#### 🏗 Create the Application
To start the deployment, you need to create the ArgoCD Application:
1.  In the ArgoCD UI, click **"New App"**.
2.  **Application Name**: `personality-engine`
3.  **Project**: `default`
4.  **Sync Policy**: `Automatic` (check "Prune Resources" and "Self Heal").
5.  **Source**:
    *   **Repository URL**: `https://github.com/ameerahaider/personality-engine-infra.git`
    *   **Revision**: `main`
    *   **Path**: `gitops-manifests`
6.  **Destination**:
    *   **Cluster URL**: `https://kubernetes.default.svc`
    *   **Namespace**: `personality-simulation-engine-app`
7.  Click **"Create"**.

### 4. Verify & Access
Once ArgoCD starts the sync, you can monitor the deployment:

#### ✅ Check Deployment Status
```bash
kubectl get pods -n personality-simulation-engine-app
```

#### 🌐 Find the Application IP
Wait for the LoadBalancer to be provisioned (this may take 2-3 minutes):
```bash
kubectl get ingress -n personality-simulation-engine-app
```
Look for the **ADDRESS** column. You can visit this IP in your browser to see the live app.

> [!NOTE]
> **Missing IP?**: If the ADDRESS is empty after several minutes, ensure the NGINX Ingress Controller is installed. Run `terraform apply` in the `terraform/` directory to deploy the `ingress-controller.tf` configuration.

#### 📜 Check Application Logs
To debug or monitor the components:
*   **Backend API**:
    ```bash
    kubectl logs -l app=backend -n personality-simulation-engine-app --tail=50
    ```
*   **Frontend Dashboard**:
    ```bash
    kubectl logs -l app=frontend -n personality-simulation-engine-app --tail=50
    ```

---

> [!TIP]
> **Secret Management**: Connection strings for the managed database are automatically generated by Terraform and stored in a Kubernetes secret named `db-credentials` within the `personality-simulation-engine-app` namespace. This keeps sensitive data out of the manifests.
